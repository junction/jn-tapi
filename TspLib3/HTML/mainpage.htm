TSP++ Version 3.053 (December 2005)

Introduction

Welcome to TSP++ Version 3.0 - the easiest way to create 3 rd party Telephony Service Providers for Windows. Please read these release notes carefully as they give additional or updated information from the documentation.

TSP++ Wizard

The TSP++ Wizard is installed as an icon in the TSP++ Start Menu folder.  You can run it by activating this icon.  This wizard will create the skeleton for a TSP using the options selected.  You can run the wizard as many times as necessary to create your TSP, however each run will create a new shell - it doesn't do round-trip code generation.

Build/Test Environment

We officially support Visual Studio.NET 2005 as the primary compiler for TSP++.   The product might not compile under older versions of Visual C++ because of its use of the new secure functions, templates and heavy STL within the code.   To ensure compatibility with your system, the library is installed in source form only - you must build the library before you use it.  This will ensure that you get a proper image using the SDK configuration on your development workstation.  

In order to build the library, you must load the project file (.VCPROJ) from the SRC and UISRC directory and compile them for all configurations (Debug, DebugUnicode, Release, ReleaseUnicode).  Make sure you perform this step for both the SRC and UISRC projects - both are necessary.  When the compilations are completed, you should have a set of 8 library files in the LIB directory.  This library directory will need to be included in any TSP project you create.  In addition, the INCLUDE directory will also need to be part of the TSP project.  You can do this in the project settings for your TSP, or you can add the INCLUDE and LIB directories to the Visual Studio project directories.

TAPI 2.x

We recommend that you do the majority of your development and testing in the Windows XP or Windows Server 2003 checked build environment with full debug symbols installed.  This will allow you to trace the interactions between your TAPI applications, the TAPI system, and your TSP.  If you are using NT4, w e also recommend updating to Windows NT Service Pack 4 or better (at this writing, Service Pack 6a is available) in order to get the most stable version of TAPI.  If you are using Windows 2000 or better, any service pack should suffice.

TAPI 3.0

If you are developing for TAPI 3.0, you will need to install and run a minimum of Windows 2000.

Note that TSP++ assumes TAPI 3.x support in terms of the internal data structures - this does not mean that you must support the TAPI 3.x features in your TSP - it simply means that the TAPI.H and TSPI.H must have TAPI3 structures.  The TSP library will automatically remove any TAPI 3.x elements when running under a legacy version of TAPI.

Provided Samples

Both of the provided TSP samples (JTSP and DSSP) were designed to run under Windows 2000 or higher.  They have not been tested under Windows 95/98 and will not run in the shipped pre-built form. If you decide to install it under Windows 95 or Windows 98, you must re-compile the source code to utilize ANSI.

The ACD simulator, JPBX, requires Internet Explorer 4.0 (or a newer version of COMCTL32.DLL) to run properly because it uses some of the new control classes for the user-interface. It runs fine under Windows 95 and Windows 98 as long as the MFC42xx.DLL files are from Microsoft Visual C++ 6.0 and the latest COMCTL32.DLL is in the WINDOWS\SYSTEM directory. If the sample refuses to load, it is most likely due to a mismatch of DLL files on the system. Check the MFC42xx.DLL files and make sure they are dated at least May/June of 1998. Also, make sure the system has Microsoft Internet Explorer 4.0 (or later)
